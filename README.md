# Оцінка утримання користувачів (Retention Rate) методами Google Sheets та SQL на основі когортного аналізу (Cohort Analysis)
## У цьому проєкті я проаналізувала утримання користувачів для різних джерел залучення за допомогою когортного аналізу.
## Дані

Проєкт використовує дві таблиці з бази даних:

### cohort_users_raw
Містить інформацію про користувачів:
- user_id — унікальний ідентифікатор користувача
- signup_datetime — дата реєстрації (потребує очищення)
- signup_source — канал залучення
- promo_signup_flag — тип залучення (промо / органічний)

### cohort_events_raw
Містить інформацію про дії користувачів:
- user_id — ідентифікатор користувача
- event_datetime — дата події (потребує очищення)
- event_type — тип події (login, purchase тощо)
- revenue — дохід з події (якщо є)
## Інструменти:
- SQL
- DBeaver
- Google Sheets
## Основні етапи 
- Очищення та приведення дат до єдиного формату
- Об'єднання даних з двох таблиць за допомогою Join
- Формування когорт за місяцем реєстрації  
- Розрахунок активності користувачів у динаміці (month offset)  
- Фільтрація тестових та некоректних подій  
- Експорт підготовленого датасету в Google Sheets  
- Побудова когортних таблиць за допомогою pivot tables  
- Розрахунок Retention Rate  
- Створення інтерактивного фільтра (Slicer) для сегментації користувачів 
## Висновки
Аналіз когорт показав суттєву різницю в поведінці користувачів залежно від каналу залучення. Органічні користувачі демонструють значно вищий та стабільніший рівень утримання на всіх етапах життєвого циклу. Вже на першому місяці їх retention становить 73–87% проти 46–61% у промо-користувачів, а в подальшому зберігається на високому рівні без різких падінь.
Користувачі, залучені через промо-активності, демонструють більш виражене зниження активності з часом, що свідчить про нижчу довгострокову залученість у продукт. Таким чином, органічний канал залучення забезпечує більш лояльну та стабільну аудиторію, тоді як промо-кампанії ефективні переважно для короткострокового зростання.
